1. No contexto do sistema de controlo de versões Git, qual a diferença entre os comandos clone e pull?


O comando 'clone' serve para fazermos uma cópia (clone) de um certo repositório git, por exemplo, o do projeto 
de grupo de LS para uma determinada diretoria e trabalharmos a partir dela, tal como fizemos para esta avaliação;

O comando 'pull' serve para requisitar os commits mais recentes dados para um repositório git que ainda
não se encontram no nosso repositório local de forma a atualizá-lo.
